# Briefs

User-facing research briefs produced by applying the skills in this repo to the holder's actual portfolio questions. Unlike `reports/` (which is gitignored and holds per-run script output), `briefs/` is **committed** so each brief is directly readable on GitHub without cloning.

## Structure

```
briefs/
├── ai-infra/          # AI-infrastructure opportunity briefs
│   ├── README.md      # index for this topic
│   └── YYYY-MM-DD_*.md
└── <future topics>/
```

## Conventions

- Filename: `YYYY-MM-DD_<topic>.md` (sortable, date-first).
- Each topic folder maintains its own `README.md` index.
- Every brief cites the skills in `skills/` whose frameworks it applies — no claim is made that isn't grounded in an existing skill or an explicit disclaimer.
- Briefs are advisory, not financial advice.
